vue는 Progressive Framework라는 개념으로 만들어졌다. Progressive Framework란 필요한 기능을 필요한 만큼만 도입해서 사용할 수 있으며, 필요에 따라 기능을 추가하여 사용할 수 있다는 것이다.

그러다 보니 vue를 사용하는 가장간단한 방법은 vue.js파일을 페이지에 직접 로드하여 사용하는 방법이다.(참조 : https://www.codeproject.com/Tips/1271379/Simple-ASP-NET-CORE-2-2-App-plusVue-JS) 하지만 프로젝트의 규모가 커지면 스크립트 파일이 매우 많아지게 되고 결국 webpack등을 이용해서 스크립트를 빌드하기 위한 방법을 선택하게 된다. 일반적으로 이 과정에서 직접 vue.js를 페이지에서 로드하기 보다는 npm등을 통해 패키지로 설치한 뒤 import를 통해 각 모듈을 불러와서 사용하게 된다.

​

그런데 asp.net core등의 웹 서버 프레임워크와 함께 개발을 진행하는 경우 제대로 빌드설정을 해주지 않으면 vue.js가 포함된 프론트엔드와 웹 서버 어플리케이션을 각각 빌드해야 하는 단점이 생긴다. 이 문서에서는 이러한 문제를 해결하기 위해서 asp.net core와 vue.js를 같이 빌드할 수 있도록 설정하는 방법에 대해 알아본다.

- asp.net core 2.1 + Vue 설정하기

​

asp.net core 2.1과 2.2의 경우는 거의 유사하지만 2.1은 vs2019로 2.2는 dotnet 커맨드와 vs code로 작업하는 방법을 설명하기 위해 각각 따로 설명한다.

vue-dotnet2.1이라는 이름으로 새 프로젝트를 생성한다.

이번에는 asp.net core 2.1을 선택하고 프로젝트 템플릿은 React.js를 선택한다. 그리고 오른쪽의 “HTTPS에 대한 구성”은 선택을 해제한다.

React.js는 기본 MVC템플릿과는 다르게 이미 ClientApp폴더가 만들어져 있으며, 여기에 React.js와 관련된 코드가 포함되어 있다. 우리는 vue를 사용할 것이므로 여기에 있는 내용을 모두 Vue와 관련된 코드로 교체할 것이다. 우선 vue와 매끄러운 연동을 위해서 NuGet패키지 매니저를 통해 VueCliMiddleware를 설치한다.

솔루션 탐색기에서 “종속성” 항목에서 우클릭하고 NuGet 패키지 관리로 넘어간다.

VueCli로 검색하여 나타나는 VueCliMiddleware를 선택하고,

오른쪽 뷰에서 버전을 2.1.1로 선택하고 설치한다.

패키지가 설치되고 나면 Startup.cs를 다음과 같이 수정한다.

using Microsoft.AspNetCore.Builder; using Microsoft.AspNetCore.Hosting; using Microsoft.AspNetCore.Mvc; using Microsoft.AspNetCore.SpaServices.ReactDevelopmentServer; using Microsoft.Extensions.Configuration; using Microsoft.Extensions.DependencyInjection; //----- 1. using문 추가 using VueCliMiddleware; namespace vue_dotnet2._1 { public class Startup { public Startup(IConfiguration configuration) { Configuration = configuration; } public IConfiguration Configuration { get; } // This method gets called by the runtime. Use this method to add services to the container. public void ConfigureServices(IServiceCollection services) { services.AddMvc().SetCompatibilityVersion(CompatibilityVersion.Version_2_1); // In production, the React files will be served from this directory services.AddSpaStaticFiles(configuration => { //----- 2. 배포용 빌드 폴더를 변경 configuration.RootPath = "ClientApp/dist"; }); } // This method gets called by the runtime. Use this method to configure the HTTP request pipeline. public void Configure(IApplicationBuilder app, IHostingEnvironment env) { if (env.IsDevelopment()) { app.UseDeveloperExceptionPage(); } else { app.UseExceptionHandler("/Error"); } app.UseStaticFiles(); app.UseSpaStaticFiles(); app.UseMvc(routes => { routes.MapRoute( name: "default", template: "{controller}/{action=Index}/{id?}"); }); app.UseSpa(spa => { spa.Options.SourcePath = "ClientApp"; if (env.IsDevelopment()) { //----- 3. 디버깅시 실행할 개발 커맨드 변경 spa.UseVueCli(npmScript: "serve"); } }); } } }

그리고 .csproj파일의 내용중 맨 아래쪽에 위치한 ItemGroup태그의 내용중 빌드 결과물이 위치하는 폴더 명을 build에서 dist로 수정한다.

<ItemGroup> <DistFiles Include="$(SpaRoot)dist\**" /> <ResolvedFileToPublish Include="@(DistFiles->'%(FullPath)')" Exclude="@(ResolvedFileToPublish)"> <RelativePath>%(DistFiles.Identity)</RelativePath> <CopyToPublishDirectory>PreserveNewest</CopyToPublishDirectory> </ResolvedFileToPublish> </ItemGroup>

이제 vue를 위한 설정은 모두 끝났으므로 vue코드를 생성하기 위해 커맨드 창을 열어 프로젝트가 위치한 폴더로 이동한다. 그리고 npm을 이용하여 vue와 vue-cli-service를 설치한다.

> npm -g i @vue/cli @vue/cli-service

그리고 clientapp이라는 이름으로 vue프로젝트를 생성한다.

> vue create clientapp

그러면 vue프로젝트 생성을 위한 조건을 물어보는데 위아래 방향키와 엔터키로 선택이 가능하다.이미 폴더가 존재하므로 덮어쓰기를 선택한다.

기타 설정은 간단한 시작을 위해서 기본 값을 선택한다.

조건을 선택하고 잠시 기다리면 vue앱 생성이 완료된다.

vue-cli가 대문자가 포함된 앱 이름을 허용하지 않으므로 앱 이름을 ClientApp이 아닌 clientapp으로 설정하였고, 따라서 폴더 역시 clientapp으로 생성되었다. 윈도우에서는 대소문자를 구분하지 않으므로 상관이 없으나 나중에 Ubuntu등의 리눅스에 배포하기 위해서는 대소문자를 구분하여야 한다. 다음 커맨드를 실행하여 폴더 이름을 ClientApp으로 변경한다.

> ren clientapp ClientApp

- asp.net core 2.2 + Vue 설정

​

이번에는 Visual Studio Code와 dotnet커맨드를 통해서 asp.net core 2.2와 vue를 연동해보겠다. Visual Studio Code가 없는 경우 https://code.visualstudio.com/ 로 이동하여 설치한다. 먼저 커맨드를 통해서 vue-dotnet2.2 폴더를 생성하고 “code .”명령어로 vs code를 실행해둔다.

그리고 커맨드창에서 ‘dotnet new react --no-https’명령어로 react템플릿 프로젝트를 생성한다.

dotnet new react --no-https

그리고 “dotnet add package VueCliMiddleware”명령으로 패키지를 설치한다.

dotnet add package VueCliMiddleware

- git 에서 소스 가져오기

​

Ubuntu에 앱을 배포하기 위해서는 작성된 코드를 git저장소에 올린 후, ubuntu에서 저장소의 코드를 복제해야 한다. 직접 만든 저장소를 활용하거나 이 문서에서 활용하는 https://github.com/netscout/dotnet.git 를 활용하여 진행한다.

루트 폴더에서 apps폴더를 만들고 apps폴더로 이동하여 다음 커맨드로 git 저장소를 복제한다.

> mkdir apps > cd apps > git clone https://github.com/netscout/dotnet.git

그리고 vue-dotnet2.0 폴더로 이동하여 npm 패키지를 설치하고 배포용 빌드를 생성한다.

> cd dotnet/ > cd vue-dotnet2.0 > cd vue-dotnet2.0 > npm i > dotnet publish -c Release -o ~/aspnetcore/vue-dotnet2.0

빌드가 에러 없이 완료되면 다음과 같은 출력이 되며, 맨 마지막 줄에 빌드된 앱이 배포된 위치가 표시된다.

ASP.NET Core에서 기본 제공되는 Kestrel을 통해 빌드된 앱을 실행할 수 있다.

> dotnet ~/aspnetcore/vue-dotnet2.0/vue-dotnet2.0.dll

Kestrel이 실행되면서 http://localhost:5000에서 앱을 호스팅하고 있다.

윈도우에서 브라우저를 열어 위 주소로 접속하면 vue앱이 실행되는 것을 확인할 수 있다.

2.1 버전과 2.2 버전 역시 이와 동일하지만 미세하게 다른 점이 있다. 프로젝트 루트 폴더에서 npm i를 실행하는 것이 아니라 ClientApp폴더 안에서 npm i를 실행한다는 점이 다르다. vue-dotnet2.2 폴더로 이동하여 다음 커맨드를 실행한다.

> cd ~/apps/dotnet/vue-dotnet2.2/ (폴더 이름이 소문자로 된 경우 변경) > mv -T clientapp/ ClientApp > cd ClientApp/ > npm i > cd .. > dotnet publish -c Release -o ~/aspnetcore/vue-dotnet2.2/> dotnet ~/aspnetcore/vue-dotnet2.2/vue-dotnet2.2.dll

그리고 브라우저를 이용해 http://localhost:5000으로 이동하면 결과를 확인할 수 있다.

- nignx로 프록시 설정하기

​

실무에서는 위 방법 처럼 Kestrel을 직접 외부에 노출시키기 보다는 nignx를 통해서 역방향 프록시를 설정한다. 즉,

출처 : https://docs.microsoft.com/ko-kr/aspnet/core/fundamentals/servers/kestrel?view=aspnetcore-2.2

위 사진 처럼 Kestrel 서버가 직접 외부의 http요청을 처리하는 것이 아니라, nginx가 요청을 받아들이고 그 요청을 다시 Kestrel에 전달하는 방식이 되는 것이다. 다음 명령어로 nginx를 설치한다.

> sudo apt-get install nginx

그리고 프록시 설정을 위해서 /etc/nginx/sites-avaliable/default 파일을 수정해야 한다. vim으로 파일을 열어서 server영역 안에 위치한 location 영역을 다음과 같이 수정한다.

> sudo vim /etc/nginx/sites-available/default (i를 눌러 편집모드로 전환)
server { listen 80; location / { proxy_pass http://localhost:5000; proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection keep-alive; proxy_set_header Host $http_host; proxy_cache_bypass $http_upgrade; } }

위 내용을 작성후 esc를 누르고 커맨드 모드로 전환한 후 “:wq!” 명령어로 편집한 내용을 저장한다. 그리고 다음 명령어로 nginx를 재시작하고, vue앱을 실행한다.

> sudo service nginx restart > dotnet ~/aspnetcore/vue-dotnet2.2/vue-dotnet2.2.dll

그리고 브라우저에서 http://localhost로 접속하면 nginx의 프록시를 통해 내부적으로 localhost:5000에서 실행 중인 앱에 접근하게 된다.
